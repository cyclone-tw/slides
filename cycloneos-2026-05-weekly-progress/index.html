Current Architecture

目前 CycloneOS 的定位：雲端控制台 + 本地工具執行器 + 外部知識來源

Dashboard 不應直接 SSH 到 Mac mini，也不應把 Google Drive local sync path 當 production database。它的責任是建立任務、顯示狀態、收斂資料流、控制 approval。

Agent Boundaries

每個 Agent 的工作邊界：誰負責想，誰負責做，誰負責管制

這是後續架構穩定的關鍵。CycloneOS 不是把所有能力塞進同一隻 agent，而是用 Dashboard 控制任務與權限，用不同 agent / executor 負責不同層。

Cloud Broker

OpenAB

負責 Dashboard 內的 knowledge chat、task creation、Codex CLI runtime、OpenClaw command broker。它可以思考、草擬、建立任務，但不能繞過 approval 直接寫外部系統。

  • 適合：repo 分析、任務拆解、preview-only 草稿、broker task。
  • 不適合：直接跑 Mac local tool、任意下載、未審核 publish。
Edge Executor

OpenClaw / Draco

跑在 Mac mini，本地工具由它執行。Dashboard 不直接 SSH 執行，而是透過 OpenAB 建立 allowlisted task，再由 Draco worker claim / execute / 回報結果。

  • 適合：yt-dlp、Whisper、python-docx、本機檔案、Mac automation。
  • 限制：每個 action 要 allowlist；高風險 action 需要 approval。
Specialist Agent

Hermes / Aurelion

適合放在 Research、Felo、圖片、簡報、Deep Research 類工作。建議用 adapter 方式接入，不把 Hermes 程式碼混進 CycloneOS 核心，保留未來升級 Hermes 的彈性。

  • 適合：web research、Felo skills、圖片 / 簡報生成。
  • 下一步：定義 Hermes adapter contract 與 task/event format。

Dashboard 的責任

提供使用者操作介面、source picker、Pipeline editor、status / logs、approval gate、audit trail。Dashboard 是控制台，不是萬能執行器。

Codex / Claude Code 的責任

目前是開發協作 agent，不應被視為 CycloneOS runtime 的正式產品功能。開發 session log 若要保留，應轉為 engineering audit 或 PR review log。

Target Workflows

下一階段要讓資料流真正跑起來的三條主線

這三條是使用者價值最高的主線：會議紀錄、YouTube / research、模板文件產出。它們都要共享 source、artifact、approval 與 publish 機制。

01 Source
匯入資料

Prompt、Drive file、Notion page、URL、YouTube、OpenClaw local file、Artifact ref。

02 Agent / Skill
選擇處理者

OpenAB 思考 / Hermes research / OpenClaw local tool / SkillAction。

03 Artifact
產生中間結果

Transcript、summary、docx data、markdown、slides HTML、evidence。

04 Approval
預覽與確認

外部寫入、下載、發布、Mac automation 都需要明確確認。

05 Publish
輸出與再利用

Obsidian、Drive、Notion、GitHub Pages、Discord；產物可再次當 input。

會議紀錄 / IEP / 課發會

語音、文件、圖片進來後，由 OpenClaw 或其他 agent 讀取，套用你的範本與指引，用 python-docx 產生可交付文件。

YouTube / Media / Research

先偵測字幕；無字幕才走 audio download + Whisper；後續摘要可輸出 Notion、Obsidian、簡報 HTML 或 GitHub Pages。

模板驅動文件

任何 docx template 都能搭配 Drive / Notion / prior artifact 產出申請計畫、IEP、課程計畫、教案與簡報。

Roadmap

後續 issue 與建議優先順序

建議先補使用者操作可理解性，再補資料流輸入整合，最後才擴大 Mac 工具與外部發布；否則功能會很多，但使用者不知道怎麼安全地跑。
