積分機制說明

積分榜 /leaderboard & 排名意義

  • 前 3 名 — 🥇🥈🥉 獎台展示
  • 第 4 名起 — 排行表格 — 顯示總積分、打卡天數、連續天數
  • 排名的意義 — 不是競爭，是看見彼此的努力，激勵持續投入、養成每日學習的習慣
  • 特別獎勵 — 當月最高者可獲得特別獎勵（待討論）

知識庫 vs AI 工具箱

📚 知識庫 /knowledge

  • 定義 — 分享你在學習過程中獲得的知識、心得、筆記
  • 分類 — 模板 / 最佳實踐 / Q&A / 其他
  • 範例 — 「我用 Claude 做了一個自動整理 email 的 prompt，分享模板」
  • 積分 — 投稿 +20，每讚 +5
VS

🤖 AI 工具箱 /ai-tools

  • 定義 — 分享好用的 AI 工具，或展示自己做的作品
  • 分類 — Agent / LLM / 生產力工具 / 開發工具 / 其他
  • 範例 — 推薦他人作品 +30；自己作品 +100（附 Repo 或部署網址）
  • 關鍵差異 — 知識庫 → 學到什麼；AI 工具箱 → 用什麼工具 / 做了什麼工具

許願樹 /wishlist — 互助機制

  1. 1許願 — 提出需求 — 個人需求如「我想要一個自動整理 Notion 的腳本」；或網站功能「希望知識庫可以加搜尋功能」
  2. 2認領 — 有人接手你的許願
  3. 3實作中 — 在 AI 工具箱投稿成果，並關聯許願卡
  4. 4完成 — 完成按鈕在投稿關聯後才能生效，完成他人許願 +300 分（最高獎勵）

用別人的視角打磨你的 AI 工作流，幫助別人的同時也在練功

目前已知的待改善項目

功能與 UI

  • 團隊頁面聯動 — 新增成員無法自動更新，需手動；後台需新增 emoji/代表色/自介欄位
  • 人員篩選聯動 — 知識庫/工具箱篩選清單應從後台未封存帳號聯動
  • 留言機制 — 目前只能按讚，無法對投稿留言反饋
  • Dashboard 個別化 — 知識庫/工具箱區塊應載入個人投稿與按讚收藏內容

系統與管理

  • 積分榜得分說明 — 頁面上需列出完整得分規則
  • 許願樹流程改善 — 認領後須在工具箱投稿並關聯許願卡，才能標記完成
  • 登入審核機制 — 未知 email 登入應為「待審核」，隊長確認後才指派角色
  • 後台成員管理 — 刪除/停用成員、修改顯示名稱、角色指派確認 dialog
