改造層次

已建立的邊界

  • Dashboard 是 control plane：登入、UI、審核、task history、provider API。
  • OpenAB 是 broker/agent runtime：接任務、排隊、未來管理 worker lifecycle。
  • Mac mini/OpenClaw 是 execution plane：只做 allowlisted、可追蹤的 Mac/local 動作。
  • Postgres 是 durable state：channel、message、task history 已開始落地。

刻意不做的事

  • Dashboard web request 不直接啟動 Codex/Claude 長任務。
  • Zeabur Dashboard 不依賴 /Users/*、local Drive sync、local Obsidian path。
  • legacy Discord bot 不再是 Dashboard repo runtime。
  • 舊有 native OpenAB 目前沒有被改原始碼，新 CycloneOS OpenAB 是分離候選。

新 CycloneOS OpenAB 狀態

已完成

  • upstream openabdev/openab 匯入 services/openab。
  • 加入 Dockerfile.cycloneos-openab 與 Zeabur overlay。
  • broker-only boot 可在缺少完整 bot/provider secret 時啟動。
  • /health 回傳 OK；authenticated POST /tasks 可建立 queued task。
  • Rust release build 已在 Zeabur 通過。

尚未完成

  • 尚未綁公開 domain，因此 Dashboard 還不能穩定 cutover。
  • task store 仍是 in-memory，服務重啟會遺失。
  • worker claim、heartbeat、complete/update API 尚未做。
  • broker 還沒有 enforce allowlisted action，只做基礎 type/target validation。
  • 舊 service 與測試 service 需要最後清理。
